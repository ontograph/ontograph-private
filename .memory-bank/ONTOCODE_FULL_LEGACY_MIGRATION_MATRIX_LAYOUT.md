## Evidence

- Local inventory found 88 non-target files with direct `ontocode-rs`, `//ontocode-rs`, or `CODEX_RS_ROOT` references.
- Local inventory found 125 `Cargo.toml` files and 125 `BUILD.bazel` files under the current `ontocode-rs/` workspace.
- OntoIndex semantic search for layout/build graph migration returned relevant anchors in package staging, SDK generation, Bazel labels, schema fixtures, snapshot workspace roots, and CI workflows.
- OntoIndex freshness is degraded by dirty-worktree overlay and missing embeddings, so implementation must rerun exact OntoIndex impact/context on edited code symbols before any code slice.

## Rules For Stage 1

- Rename only the workspace layout surface: `ontocode-rs/` -> `ontocode-rs/`.
- Do not rename public CLI commands, helper binaries, package identities, env vars, state roots, protocol names, telemetry names, generated type names, or historical audit text in this stage.
- Keep Cargo package names and Bazel target names unchanged unless a path-only BUILD label update requires it.
- Any remaining `ontocode-rs` match after Stage 1 must be classified as historical, compatibility, generated, or deferred.
